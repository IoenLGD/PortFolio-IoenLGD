PRTG Network Monitor – Mode Opératoire Complet

Réalisé en stage BTS SIO SISR – Saint Joseph la Salle, Lorient
Par : ELBASAN Kendal, LE GOURLAY-DENIS Ioen, DECOURT Enzo
Session : janvier/mars 2026

1. Qu'est-ce que PRTG ?

PRTG (Paessler Router Traffic Grapher) est un logiciel de supervision réseau tout-en-un sans agent. Il surveille en temps réel l'état, la disponibilité et les performances d'une infrastructure IT (routeurs, switchs, serveurs, pare-feu, NAS, etc.).

Objectifs principaux :

  • Détecter les pannes ou anomalies avant impact utilisateur
  • Analyser performances (CPU, RAM, bande passante, latence...)
  • Prévenir les incidents via alertes (email, push, son...)
  • Produire graphiques, rapports et tableaux de bord

Utilisé en entreprise, établissements scolaires, hôpitaux, réseaux personnels avancés et projets BTS SIO.

2. Fonctionnement de base

PRTG repose sur une hiérarchie :

  • Groupe : catégorie (ex. Réseau, Serveurs)
  • Appareil (Device) : machine/équipement (IP ou nom DNS)
  • Capteur (Sensor) : élément surveillé (Ping, CPU, trafic...)

Protocoles utilisés : Ping, SNMP, WMI (Windows), SSH (Linux), HTTP/HTTPS, NetFlow/sFlow...

3. Installation de PRTG (sur Windows)

PRTG s'installe uniquement sur Windows (10/11 ou Server 2016+).

  1. Télécharger la version Free (100 capteurs) sur site officiel Paessler.
  2. Lancer l'installateur (.exe).
  3. Accepter la licence, choisir dossier (défaut OK).
  4. Entrer email et mot de passe admin fort.
  5. Laisser ports par défaut → Installer (5-10 min).

Premier accès : https://localhost (ou IP du serveur) → Utilisateur : prtgadmin / mot de passe défini.

4. Découverte automatique du réseau

  1. Dans l'arborescence → Clic droit sur "Probe locale" → "Ajouter un groupe de découverte automatique".
  2. Dans "Informations d'identification" : entrer plage IP (ex. 192.168.1.1 - 192.168.1.254).
  3. Lancer la découverte → PRTG scanne et crée automatiquement appareils + capteurs de base (Ping, SNMP...).

5. Mise en place de capteurs essentiels

  • Capteur Ping (sur tous les équipements) :
    1. Clic droit sur appareil/groupe → Ajouter capteur.
    2. Rechercher "Ping" → Sélectionner "Ping standard".
    3. Valider → Vérifie disponibilité (vert = OK, rouge = hors ligne).
  • Capteur Trafic (ports fibres optiques d'un switch) :
    1. Activer SNMP sur le switch (communauté "public" ou privée).
    2. Sur l'appareil switch dans PRTG → Ajouter capteur → "SNMP Traffic".
    3. Sélectionner le port fibre → Valider → Surveille bande passante entrante/sortante/totale.

6. Configuration des alertes et notifications par email

  1. Aller dans "Configuration" → "Notifications" → Créer notification "Email".
  2. Configurer serveur SMTP (ex. Gmail : smtp.gmail.com, port 587, TLS, identifiants).
  3. Créer déclencheur sur capteur :
    • Pour Ping : "Down" → Sévérité Haute → Notification email.
    • Pour Trafic : "Speed In/Out/Total > seuil" (ex. 90% saturation) → Notification email.
  4. Tester l'alerte pour vérifier réception.

7. Bonnes pratiques et résumé

  • Toujours ajouter un capteur Ping par appareil
  • Nommer clairement groupes/appareils/capteurs
  • Limiter capteurs inutiles (PRTG Free : max 100)
  • Utiliser avec GLPI pour tickets automatisés

Phrase clé : PRTG surveille en continu, alerte avant panne et anticipe les problèmes réseau.
